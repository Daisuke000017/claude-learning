Anthropicが公式提供するChrome拡張機能です。Claude Codeと連携し、ブラウザのWebページを自動で読み取り・操作できます。Claude Code から「このURLの内容を読んで」と指示すると、Claude for Chrome がブラウザを通じてページ内容を取得します。
STEP-BY-STEP GUIDE

再現手順 ─ 実際にやってみよう

以下の手順に沿って、Claudeに指示するだけで同じ成果物を作れます。

1

Webページのコンテンツ分析・抽出

Claude for Chrome でWebページを自動読み取り

事前準備

Claude for Chrome をセットアップする

  1. Chrome Web Store から「Claude」拡張機能をインストール
  2. Chromeの右上の拡張機能アイコンからClaudeを開き、ログイン
  3. ターミナルで claude コマンドを起動（Claude Code）
  4. Claude Code が Chrome と自動連携されます
Claude for Chrome は Pro / Max / Team / Enterprise プランで利用可能。現在ベータ版で、Google Chrome と Microsoft Edge に対応しています。
Claudeへの指示

コースの内容を分析してもらう

Claude Code のターミナルで、以下のように指示します：

Claude Code に入力するプロンプト
このコースの内容を初心者向けに資料に詳細にまとめてください https://anthropic.skilljar.com/ai-fluency-framework-foundations
Claudeが自動でブラウザを操作し、コースの各レッスンページを順番にアクセス。テキストを読み取り、学習目標・キーポイント・演習内容を整理したノートを生成します。
裏側で起きていること

Claudeの自動処理フロー

  1. Claude for Chrome がブラウザでコースページにアクセス
  2. ページのテキストコンテンツを自動で読み取り
  3. 「次のレッスン」リンクをクリックして全レッスンを巡回
  4. 各レッスンの内容をMarkdown形式のノートに整理
  5. コース全体の構成を分析し、セクション構造を設計
ログインが必要なページの場合、事前にブラウザでログインしておく必要があります。Claude for Chrome は開いているブラウザのセッションを使ってアクセスします。

Claudeへの指示

Step 1 でコンテンツ分析が終わった後、そのまま同じ会話で以下を指示します：

Claudeが HTML / CSS / JavaScript のコードを自動生成し、ファイルとして保存します。フェードインアニメーション、スマホ対応、アイコン表示など、モダンなWebページが完成します。
自動で含まれる機能

3

PowerPointスライドの作成

同じ内容をPPTX形式の資料に自動変換

Claudeへの指示

一言で完了

Claude Code に入力するプロンプト
PowerPoint資料も作成して
ClaudeがNode.jsスクリプトを自動で作成・実行し、PptxGenJSライブラリを使ってPowerPointファイル（.pptx）を生成します。タイトルスライド・目次・解説・まとめが含まれます。
裏側で起きていること

Claudeの自動処理フロー

  1. PptxGenJS（PowerPoint生成ライブラリ）を自動インストール
  2. スライド生成用のNode.jsスクリプトを作成
  3. 統一デザイン（カラーパレット・フォント・レイアウト）を適用
  4. スクリプトを実行し、.pptx ファイルを出力
生成されたPPTXファイルは Microsoft PowerPoint、Google Slides、Keynote で開けます。
4

GitHubへのアップロード・公開

ファイルをGitHubにアップしてWebサイトとして公開

事前準備

GitHubリポジトリを作成しておく

  1. github.com にログイン
  2. 右上の「+」→「New repository」をクリック
  3. リポジトリ名を入力（例：claude-learning）
  4. 「Public」を選択し「Create repository」をクリック
Claudeへの指示

リポジトリ名を伝えるだけ

Claude Code に入力するプロンプト
GitHubにhtmlデータをアップロードしたいです ユーザー名/claude-learning
Claudeが自動でGit操作（clone → add → commit → push）を実行し、ファイルをGitHubにアップロードします。目次ページ（index.html）も自動で作成されます。
サイト公開

GitHub Pages を有効にする（手動）

アップロード後、以下の設定でWebサイトとして公開します：

  1. GitHubリポジトリの Settings タブをクリック
  2. 左メニューの Pages をクリック
  3. Source: Deploy from a branch を選択
  4. Branch: main / /docs を選択
  5. Save をクリック
  6. 数分後に https://ユーザー名.github.io/リポジトリ名/ で公開
GitHub Pages は Public リポジトリ でのみ無料で利用できます。Private リポジトリの場合は GitHub Pro（有料）が必要です。

TIPS

Claudeを使いこなすコツ

今回のワークフローで学んだ、AIを最大限活用するためのポイントです。

自然な日本語でOK

プログラミング知識不要。「〜を作って」「〜にアップして」で伝わります

段階的に依頼する

一度に全部ではなく、分析→資料→公開と段階的に進めると精度が上がります

結果を確認して修正

出力を確認しながら「ここを直して」と調整。対話的に品質を高められます

実践的なヒント

  • 参考を見せる：「このページと同じデザインで」と既存ファイルを指定すると、統一されたデザインになります
  • 対象者を伝える：「初心者向けに」「エンジニア向けに」など読者層を明示するとレベル感が調整されます
  • 具体的なURLを渡す：分析してほしいページのURLを直接貼り付けるのが最も確実です
  • 前のコンテキストを活用：同じ会話の中で追加依頼すると、前の指示内容を踏まえて作成してくれます
  • 途中経過を確認：大きなタスクは途中で「ここまでの進捗を見せて」と確認するのがおすすめです
