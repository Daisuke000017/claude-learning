Lesson 1

Coworkとは？

Claude Codeと同じアーキテクチャ上に構築された、非技術者のためのエージェント型作業ツールです。テキストを貼り付けて回答を得る従来のチャットとは根本的に異なります。

従来のチャット

  • テキストを入力して回答を得る
  • 1回のやり取りで完結
  • 外部ツールへの接続なし
  • ファイル操作は不可
  • ユーザーがコピペで橋渡し
VS

Cowork（エージェント型）

  • タスクを委任して成果物を受け取る
  • 複数ステップを自律的に実行
  • Gmail・Drive・Slack等と直接接続
  • ファイルの読み書き・作成が可能
  • ツール間を自動的に橋渡し

Lesson 1

チャットとの違い：3つの柱

Coworkは Plan → Execute → Connect の3ステップで動作します。

1

Plan（計画）

タスクを受け取ると、まず実行計画を策定。サブタスクに分解し、不明点があれば質問します。

2

Execute（実行）

ファイル読み書き、コード実行、Web検索など必要なツールを使い、並列処理（Subagent）も活用して実行します。

3

Connect（接続）

Gmail・Drive・Slack・Calendarなど、あなたが普段使うツールと直接連携し、成果物を適切な場所に配置します。

Lesson 1

Coworkの6つのコア機能

Coworkが実際にできることの全体像です。

Connectors（接続）

メール、Slack、Google Drive、Calendar など既存ツールと接続。データが自動的に流れ込みます。

File Operations（ファイル操作）

PDF・Excel・PPTXの読み書き、フォルダ整理、ファイル変換などをネイティブ形式で実行します。

Plugins（プラグイン）

Skills + Connectors + Subagents をバンドルした拡張パック。職種・ドメイン別に専門知識を追加できます。

Scheduled Tasks（定期実行）

任意のタスクを自動で定期実行。/schedule コマンドで毎朝のレポート作成なども設定可能です。

Subagents（並列処理）

独立した部分タスクを複数の子エージェントに分割し、同時並行で処理します。5社分の分析も一度に実行。

Local Computation（計算）

Pythonなどのコードをローカル環境で直接実行し、データ分析やグラフ作成を行います。

Lesson 3

タスクループ：基本の4ステップ

Coworkとの作業は、常にこの4ステップで回ります。

Describe
タスクを説明する

何を見て、何を作り、どこに保存するかを伝えます

Clarify
質問に答える

Coworkが不明点を質問するので、方針を明確にします

Execute
離席 or 見守る

進捗パネルで確認しつつ、Coworkが作業を実行します

Review
成果物を確認

完成ファイルをレビューし、修正があればフィードバック

実践例：散らかったフォルダ → 完成プレゼン

散在ファイル → 内容分析 → 構成決定 → スライド作成 → 完成プレゼン

会議メモ・チェックリスト・タイムライン・メールスレッドが散在するフォルダから、リーダーシップ向けプレゼンを自動生成する例です。

Lesson 3

良いプロンプトの3要素

Coworkへの指示は「何を見るか」「何が欲しいか」「どこに保存するか」の3要素で構成しましょう。

Input
何を見るか + Output
何が欲しいか + Where
どこに保存するか

GOOD

「今週決算報告した3社の議事録がフォルダにあります。アナリストレポートも一緒に。3社それぞれの業績サマリーと、共通テーマを比較するスライドを作成して。チームの共有DriveのQ1フォルダに保存してください。」

BAD

「決算の資料をまとめて。」

知っておくと便利な仕組み

Subagents（並列処理）

独立した部分が複数ある場合、別々のワークストリームで同時に処理。5社分のレポートを5つ同時に分析できます。

進捗パネル

サイドバーに進捗がリアルタイム表示。各ステップの完了状況を確認できます。長時間タスクでも安心。

自動リトライ

エラーが発生しても自動で別のアプローチを試みます。接続切れなどの一時的問題にも対応。

承認ステップ

メール送信やファイル削除など重要な操作は、実行前に必ずユーザーの承認を求めます。

Lesson 4

コンテキスト（文脈）の設定

Coworkはタスクごとに記憶がリセットされます。毎回説明し直さないために「プロジェクト」と「Instructions」を活用しましょう。

Global Instructions（グローバル指示）

全プロジェクト共通のルール。文体・署名・日付形式など。

Project Instructions（プロジェクト指示）

特定プロジェクト用のルール。関係者・用語・ブランドガイドなど。

Folder / File Context（ファイル文脈）

プロジェクトにアタッチしたフォルダ・ファイルが自動的にコンテキストに。

Instructionsに書くべき4項目

Who's involved（関係者）

名前・役割・連絡先を書いておくと「Rachelに送って」が通じるようになります。

例: Rachel - PM (rachel@example.com)

How it should look（見た目）

ブランドカラー、フォント、テンプレートなどのデザインルール。

例: 見出しはゴシック体、アクセント色 #02C39A

Standards（基準）

命名規則、日付形式、ファイル構成ルールなど。

例: ファイル名は YYYY-MM-DD_タイトル

Boundaries（制限）

触ってはいけないフォルダ、送信してはいけない宛先、避けるべき操作など。

例: /archive/ フォルダは変更禁止

確認＆有効化

元データ

分析・整理・変換

Lesson 8

リサーチ＆大規模分析

Coworkが特に力を発揮する3つの条件があります。

V

Volume（大量）

数十〜数百のファイルを一度に分析。人間なら数日かかる作業を短時間で。

P

Parallelism（並列）

Subagentで複数ソースを同時に調査。競合5社の分析も並行実行。

I

In-place（現地保存）

結果をそのままDriveやSlackに配置。分析→レポート→共有がワンストップ。

「まとめて」ではなく「シグナルを聞く」

最も有用なプロンプトは「全部まとめて」ではなく、自分が1週間かけて答えるような質問です。

避けるべき指示

「議事録をまとめて」

効果的な指示

「過去3ヶ月の取締役会議事録から、繰り返し議論されているが未解決の課題トップ5を抽出し、各課題について最初に言及された日・関与メンバー・最新ステータスを表にまとめて」

Lesson 9

安全設計と権限管理

Coworkは4層の安全設計で構築されています。

1

隔離環境（Sandbox）

OS本体とは分離された環境で実行。システムファイルへのアクセスは不可。

2

承認ステップ（Approval）

メール送信・ファイル削除・外部送信など重要操作は実行前にユーザー承認が必要。

3

権限スコープ（Scope）

接続ツールごとに読み取り/書き込み権限を個別設定。最小権限の原則。

4

監査ログ（Audit）

全操作のログが記録され、管理者が確認可能。Enterprise向け詳細監査も対応。

使用量管理とモデル選択

関連タスクをまとめる

セッション開始にはオーバーヘッドあり。関連する複数タスクは1セッションで実行すると効率的。

スコープを明確に

「全ファイル調べて」より「この3ファイルを分析して」の方が消費量を節約できます。

モデルを使い分ける

簡単なタスクには軽量モデル、複雑な分析には高性能モデルを使い分けて最適化。

Lesson 10

よくある問題と対処法

困ったときのトラブルシューティングガイドです。

「ワークスペース準備中」が長い

Coworkが環境を準備中。通常のチャットより時間がかかる場合があります。特に初回や大きなフォルダの場合はそのまま待ちましょう。

ツールの接続が切れた

Settings → Connectors から再認証してください。トークンの有効期限切れが原因のことが多いです。

出力ファイルの品質が低い

プロンプトにInput / Output / Whereの3要素を含めているか確認。スキルにブランドガイドラインを設定すると品質が安定します。

使用量の上限に達した

プラン上限に到達すると一時的に利用制限されます。スコープを絞る、タスクをまとめるなどの節約策を検討してください。
